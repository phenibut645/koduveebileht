MVP

Minimum Viable Product is a test version of a product, service or service with a minimum set of functions (sometimes even one), which is of value to the end user.

MVPs are designed to test the hypothesis and test the viability of the intended product, how valuable it will be and what it will be required in the market.

The results of the minimum-viable test and the feedback from the target audience help to understand whether the project should be developed further, what changes should be made to the strategy and what should be left in its original form.

The usefulness of MVP development is evidenced by examples of large companies so far.

For example, Daniel Eck and Martin Laurentson launched a small service in 2006 with a single function, the music streaming.

Today, their product, Spotify, is valued at $21 billion, collaborates with large recording studios, and has 50 million active audiences.

In 2008, when the rental of a hotel or accommodation during the trip was a major problem, two enthusiasts decided to approach the issue in an unusual way and to fax their apartment.

In fact, it's also the MVP that tested the main function.

The experiment has shown that the product will be demanded, and Airbnb is now one of the largest short-term rental sites.

Git

Et luua kaug-repositoorium (remote repository) ja ühendada see oma lokaalse repositooriumiga, tuleb järgida järgmisi samme. Siin on lihtne juhend Git-i kasutamiseks:

1. Loo kaug-repositoorium (GitHub, GitLab, Bitbucket jne)

  • Loo konto näiteks GitHubis või GitLabis.
  • Loo uus repositoorium veebiliideses ja salvesta see link (nt link).

2. Initsialiseeri lokaalne Git-repositoorium

Kui sul ei ole veel lokaalset Git-repositooriumit, siis ava terminal ja kasuta järgmisi käske:

git init

See loob .git kausta, mis muudab sinu kausta Git-repositooriumiks.

3. Lisa faile ja tee esimene commit

Kui oled juba faile loonud või soovid neid lisada, siis kasuta järgmisi käske:

git add .
git commit -m "Esimene commit"

Esimene käsk lisab kõik failid versioonihaldusse ja teine käsk teeb commit'i koos sõnumiga.

4. Lisa kaug-repositoorium

Nüüd, kui sul on lokaalne repositoorium ja kaug-repositoorium (nt GitHub), pead ühendama need. Kasuta järgmist käsku:

git remote add origin https://github.com/kasutaja/repo-nimi.git

Siin asenda link oma kaug-repositooriumi URL-iga.

5. Lükka muudatused kaug-repositooriumisse

Kui kõik on valmis, saad oma muudatused kaug-repositooriumisse saata järgmise käsuga:

git push -u origin master

Kui kasutad main haru, siis kasuta:

git push -u origin main

6. Kinnita ühendus

Kui kõik on õigesti tehtud, saad kontrollida, kas kaug-repositoorium on lisatud järgmise käsuga:

git remote -v

See näitab kaug-repositooriumite aadresse, mis on ühendatud sinu lokaalse repositooriumiga.

Repositooriumi sidumine

Lokaalse repositooriumi sidumiseks uue kustutatud repositooriumiga tuleb kasutada git remote add käsku. See käsk võimaldab lisada uue kaugrepositooriumi ja siduda selle lokaalse külge.

git remote add "name" "url

  • "name" — see on nimi kustutatud repositooriumisse (nt origin, upstream vms. d.).
  • "url" — see on uue kustutatud repositooriumi URL, mida soovite lisada.

Kokkuvõte

git init – Loo lokaalne repositoorium.
git add . – Lisa kõik failid versioonihaldusse.
git commit -m "message" – Tee commit koos sõnumiga.
git remote add origin URL – Lisa kaug-repositoorium.
git push -u origin main – Lükka muudatused kaug-repositooriumisse.

Need on põhilised käsud, mida kasutada kaug-repositooriumi loomiseks ja selle ühendamiseks oma lokaalse repositooriumiga.
